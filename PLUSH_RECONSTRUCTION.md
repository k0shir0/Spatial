# Lightweight plush reconstruction

The implemented plush/free-form baseline is an offline silhouette visual hull.
It uses NumPy, OpenCV and scikit-image only; no model weights, network, Torch,
GPU, catalog geometry or semantic template is involved.

## What it can recover

Visual hulls reproduce the observable outer envelope of a stationary object:
head, ears, torso, arms and legs. They cannot recover concavities that never
affect a silhouette, individual fur fibers, or surfaces hidden in every view.
The generic visual-hull route does not yet bake textures. The fitted soft-parts
fallback below now has a deterministic source-color bake with fabric/thread
normal and roughness maps; a short-fuzz shell remains optional and is not part
of the baseline asset.

## Required capture

- Keep the plush stationary and undeformed; no hands in reconstruction frames.
- Use a plain contrasting background and diffuse locked lighting.
- Keep camera distance and zoom fixed.
- Capture at least eight evenly spaced level views. Three rings (level and
  approximately +/-18 degrees) improve head/limb separation.
- A turntable with known angle stops is ideal. If angles are not known, rotate
  evenly and preserve chronological filenames.
- Put a scale marker beside—but not touching—the object if metric size matters.

Binary masks use white for object and black for background. Build an evenly
spaced level orbit with:

```bash
python scripts/build_visual_hull.py masks/ \
  --output object.glb --resolution 96
```

For multiple elevation rings, pass `--views-json views.json`:

```json
[
  {"file": "level_000.png", "yaw_degrees": 0, "elevation_degrees": 0},
  {"file": "level_045.png", "yaw_degrees": 45, "elevation_degrees": 0},
  {"file": "high_000.png", "yaw_degrees": 0, "elevation_degrees": 18}
]
```

## Measured prototype result

The regression fixture is an asymmetric plush proxy made from overlapping
ellipsoids for a body, head, unequal ears, muzzle, arms and legs. At 96^3:

| Views | Rings | Volume IoU | Recall | Carving time | Occupancy memory |
|---:|---:|---:|---:|---:|---:|
| 6 | 1 | 0.833 | 1.000 | 40 ms | 0.88 MB |
| 8 | 1 | 0.867 | 1.000 | 43 ms | 0.88 MB |
| 24 | 3 | 0.917 | 1.000 | 74 ms | 0.88 MB |

The exported eight-view mesh has 17,012 vertices, 34,020 triangles, one
connected component, zero boundary edges, zero nonmanifold edges and is
watertight. Eight iterations of shrinkage-resistant smoothing bring total CPU
time to roughly one second. The test remained near 0.87 IoU with up to ten
degrees of simulated angle error; one-pixel mask dilation preserved at least
99.4% ground-truth volume in that stress test.

The verified fixture is generated by the visual-hull tests. These numbers
validate the algorithm and resource profile, not real-plush accuracy. A real
capture is still required for end-to-end calibration; private development
captures and renders are intentionally not distributed.

## Hand-held fallback: fitted soft parts

The July 14 pink plush capture is hand-held, changes roll/elevation, deforms its
ears, and never exposes a hand-free silhouette. Both RealityKit and the generic
visual hull fail their geometry gates on it. For this weaker input, the project
now has a second route:

```bash
python scripts/build_soft_parts_asset.py config.json --output plush.glb
```

`src/local3d/soft_parts.py` builds colored superellipsoids, ellipsoids, and
curved tubes from a small JSON fit. The calibrated pink-plush result is 277,544
bytes, 7,216 vertices, 14,348 triangles, and builds in about 14 ms. Its primary
form is a source-fitted rounded loaf with the sleepy face on the body end, not a
separate spherical head. It is explicitly labeled `operator_fitted_soft_parts`
with inferred geometry. This route is suitable for a lightweight salvage
workflow, not autonomous measurement.

For a reviewed handheld asset, `src/local3d/texture_bake.py` can read fixed
source frames and extract clean fabric statistics into a 2048 atlas. Direct
photo projection can be disabled when changing
pose and deformation made same-part alignment unreliable even where masks were
clean. The conservative build uses source-measured fabric color/microdetail plus
explicit muzzle, accent, and thread palettes. It adds glTF-correct UVs, tangent
frames, restrained deterministic fiber relief, and nonmetallic material-class
roughness, then embeds all maps in a GLB and packages an ARKit-validated USDZ:

```bash
python scripts/build_textured_soft_parts_asset.py \
  /path/to/texture-bake-config.json \
  --output /path/to/plush-textured.glb
```

Appearance is explicitly reported as inferred from measured source patches
rather than recovered per-surface pixels. This is less ambitious than raw
projection but avoids the large photo panels and triangle-fan seams exposed by
real glTF viewers.
