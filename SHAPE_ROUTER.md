# Lightweight shape-family router

Status: local MVP implemented July 14, 2026.

The router classifies the geometry family represented by a sequence of reviewed
object masks. It does not classify an object brand/model, invoke an LLM, or run
a reconstruction automatically.

## What runs

- Runtime: OpenCV and NumPy on CPU.
- Saved classifier: `models/shape_router_v3.json`, approximately 21 KB.
- External/pretrained input: none. The classifier is trained locally from
  procedural binary silhouettes generated by this repository.
- Network, Torch, GPU, external checkpoints, and pretrained weights: none.
- Masks are analyzed sequentially at a bounded 512-pixel long side.

The implementation is in `src/local3d/shape_router.py`; the convenient entry
point is `scripts/shape_router.py`.

## Families

- `planar`
- `rounded_slab`
- `rectangular_prism`
- `cylinder`
- `bottle`
- `revolved`
- `free_form`

Phones and tins are both `rounded_slab`. A separate semantic hint or user choice
must select a subtype before subtype-specific geometry is added. No builder is
auto-dispatched for a generic rounded slab.

## How it works

For each binary mask, the router measures scale/translation/rotation-invariant
silhouette properties such as rectangularity, solidity, circularity, convexity,
symmetry, profile variation, end taper, rounded-rectangle fit, apparent aspect
ratio, and the representative edge-view/face-view ratio. Quantiles across
distinct silhouette-view clusters form a compact clip vector. A small
multinomial softmax model maps that vector to a shape family.

Version 3 expands thin manufactured-object coverage: portrait phone faces are
valid rounded slabs, while sharp thin paperback geometry remains a rectangular
prism rather than being collapsed into the `planar` family. Conservative
geometry gates only remove physically incompatible candidates; the output is
still advisory and review-only.

The router will return `unknown` instead of forcing a family when:

- Fewer than eight masks are usable.
- Fewer than four distinct silhouette clusters are present.
- Face and edge evidence required for a thin prism is missing.
- Too many masks touch the image boundary.
- Top probability or the top-two margin is too low.
- The predicted family changes when subsets of views are withheld.
- Significant silhouette holes, detached components, temporal jumps, or an
  extreme out-of-distribution feature distance are detected.

Even a valid family candidate currently returns `decision: review`,
`auto_route_eligible: false`, `recommended_builder: null`, and
`builder_supported: false`. Procedural training is not sufficient evidence for
production auto-routing. Real calibration must be split by physical object.

Cylinders, bottles, and other revolved objects can keep nearly the same
silhouette through a complete orbit. They therefore require capture telemetry
or the guided workflow to confirm rotation coverage before accepting the family
as a valid review candidate:

```text
--rotation-coverage-confirmed
```

Without that independent signal, stable repeated views return `unknown` even if
the raw predicted family is rotational.

## Example classification

```bash
python \
  scripts/shape_router.py classify \
  --masks-dir /path/to/reviewed-masks \
  --analysis /path/to/analysis.json \
  --mask-provenance object_specific \
  --output /path/to/shape-route.json
```

One internal real-capture smoke test (media not distributed) produced:

```text
family: rounded_slab
confidence: 0.985363
frame-subset agreement: 1.0
valid masks: 33
distinct view clusters: 7
face views: 2
edge views: 4
apparent edge/face ratio: 0.3625
candidate valid: true
decision: review
auto-route eligible: false
builder supported: false
```

The high probability is the synthetic classifier's score, not a real-world
accuracy claim. These mint masks were produced by an
object-specific green-tin process, so the report explicitly marks the result as
a smoke test rather than independent validation.

## Canonical segmentation input

For product integration, prefer a segmentation manifest containing a `frames`
array with `object_mask_path`, dimensions, checksum, timestamp, and confidence.
The router rejects manifests marked `placeholder: true` and verifies supplied
dimensions and checksums. The `--masks-dir` option is a legacy adapter for the
current same-stem PNG masks.

Generic deployment requires click-prompted or otherwise object-agnostic masks.
Color-specific masks cannot be used to measure general classifier accuracy.

## Reproduce the tiny model

```bash
python \
  scripts/shape_router.py train \
  --output models/shape_router_v3.json \
  --samples-per-family 280 \
  --validation-per-family 80 \
  --test-per-family 80 \
  --seed 7142026
```

The checked-in model used 1,960 procedural training clips, 560 validation clips
for temperature selection, and a separate 560-clip procedural test set. Raw
synthetic test accuracy was 95.89 percent. Rounded-slab test accuracy was 96.25
percent and rectangular-prism accuracy was 82.5 percent. These are software
regression metrics only. Production routing requires calibration on real clips
split by physical object, not by video frame.

## Tests

```bash
PYTHONPATH=src python \
  -m unittest -v tests/test_shape_router.py
```

The tests cover analytic circle/rectangle features, rotation and scale
invariance, low-coverage abstention, low-confidence masks, silhouette holes,
placeholder rejection, malformed model data, all seven procedural families, a
portrait-phone regression, and a thin-paperback regression. A private
real-capture regression is skipped when its local fixture is absent.

## Optional semantic hint

`native/VisionClassifyCLI` implements the separate semantic signal using Apple's
built-in `VNClassifyImageRequest`. It is learned vision inference supplied by
macOS—not an LLM—and the project bundles or downloads no weights. It processes
mask-cropped frames sequentially and may emit only `phone`, `tin`, `bottle`,
`can`, or `book` after cross-frame confidence/support gates. The book mapping is
deliberately narrow and does not reinterpret generic `document`, `paper`, or
`printed page` labels.

Its JSON contract explicitly sets `advisory_only: true`,
`dispatches_geometry: false`, and `overrides_geometry: false`. On the mint clip,
Vision mostly returned `document`, `machine`, and generic `container`; the
helper correctly abstained rather than turning `container` into an unsafe tin
guess. See `native/VisionClassifyCLI/README.md` for the contract.

In development captures, the helper correctly abstained when phone faces
produced only screen/graphic labels and book covers produced only generic
document evidence. This is the intended fail-closed behavior. No
silhouette-router probability should be claimed without an independent generic
reviewed-mask sequence.
